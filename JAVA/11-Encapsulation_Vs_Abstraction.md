# 🧠 1. What is Encapsulation

## What is it

Encapsulation means wrapping data and methods together into a single unit.

---

## Key Components

- Data hiding
- Private variables
- Getter/setter methods

---

## How to Answer (Interview Style)

Encapsulation is the process of binding data and methods together while restricting direct access to variables.

---

## Practical Example

```java
private int id;

public void setId(int id) {
   this.id = id;
}
```

---

## Common Mistakes

- Confusing encapsulation with abstraction

# 🧠 4. What is Abstraction ⭐

## What is it

Abstraction means hiding implementation details and showing only essential functionality.

---

## Key Components

- Hiding complexity
- Essential behavior
- Simplified interaction

---

## How to Answer (Interview Style)

Abstraction hides internal implementation and exposes only required functionality to users.

---

## Practical Example

```java
interface Payment {

   void pay();
}
```

---

## Common Mistakes

- Confusing abstraction with encapsulation

# 🧠 6. Abstract Class ⭐

## What is it

Class that can contain both abstract and concrete methods.

---

## Key Components

- Partial abstraction
- Method implementation
- Inheritance support

---

## How to Answer (Interview Style)

Abstract classes provide partial abstraction with both implemented and abstract methods.

---

## Practical Example

```java
abstract class Animal {

   abstract void sound();
}
```

---

## Common Mistakes

- Creating object of abstract class

---

# 🧠 7. Interface-based Abstraction ⭐

## What is it

Achieving full abstraction using interfaces.

---

## Key Components

- 100% abstraction
- Multiple inheritance
- Loose coupling

---

## How to Answer (Interview Style)

Interfaces achieve abstraction by defining behavior contracts without implementation details.

---

## Practical Example

```java
interface Browser {

   void start();
}
```

---

## Common Mistakes

# 🧠 8. Encapsulation vs Abstraction ⭐

| Feature | Encapsulation | Abstraction |
|---|---|---|
| Focus | Data hiding | Hiding complexity |
| Achieved Using | Private variables | Abstract class/interface |
| Purpose | Security | Simplicity |

---

## How to Answer (Interview Style)

Encapsulation hides data while abstraction hides implementation complexity.
