# 🧠 1. What is Collections Framework

## What is it

Collections Framework is a set of classes and interfaces used to store and manage groups of objects dynamically.

---

## Key Components

- List
- Set
- Queue
- Map

---

## How to Answer (Interview Style)

Java Collections Framework provides reusable data structures and algorithms to efficiently store, retrieve, and manipulate data.

---

## Practical Example

```java
List<String> names = new ArrayList<>();
```

---

## Common Mistakes

- Confusing Collection and Collections
- Not knowing interfaces hierarchy

---

## Expected Interview Questions

- What is Collections Framework?
- Why collections are used?
- Difference between Collection and Collections?

# 🧠 3. List Interface ⭐

## What is it

List stores ordered and duplicate elements.

---

## Key Components

- Ordered
- Indexed
- Allows duplicates

---

## How to Answer (Interview Style)

List maintains insertion order and allows duplicate values with index-based access.

---

## Practical Example

```java
List<String> list = new ArrayList<>();

list.add("Java");
list.add("Java");
```

---

## Common Mistakes

- Confusing List and Set
- Ignoring indexing

---

## Expected Interview Questions

- What is List interface?
- Does List allow duplicates?
- Difference between List and Set?

# 🧠 4. Set Interface ⭐

## What is it

Set stores unique elements only.

---

## Key Components

- No duplicates
- Unordered
- Unique values

---

## How to Answer (Interview Style)

Set is used to store unique elements and automatically removes duplicates.

---

## Practical Example

```java
Set<String> set = new HashSet<>();

set.add("Java");
set.add("Java");
```

---

## Common Mistakes

- Expecting insertion order in HashSet
- Ignoring uniqueness behavior

---

## Expected Interview Questions

- What is Set?
- Difference between List and Set?
- Which Set maintains insertion order?

# 🧠 11. Difference Between Collection Types ⭐

## ArrayList vs LinkedList

```text
ArrayList
→ Fast retrieval

LinkedList
→ Fast insertion/deletion
```

---

## List vs Set

```text
List
→ Duplicates allowed

Set
→ Unique values only
```

---

## HashMap vs HashSet

```text
HashMap
→ Key-value pairs

HashSet
→ Unique values only
```
